Step 1

Videos today have a very high frames per second rate. Processing this would be a tedious task especially considering the space that every good quality video takes. To avoid this, we have decided to consider only a certain number frames from a video.

In this Code we are considering 1 frame per second.

Step 2

We have used Inception CNN model for feature extraction. The second last layer is used to represent images as feature vectors

In density-based clustering, models search the data space for areas of varied density of data points in the data space. It isolates various different density regions and assign the data points within these regions in the same cluster. Popular examples of density models are DBSCAN and OPTICS.

Step 3

Object detection for videos generates meta-data which can help us optimize our search.

Hence, each cluster gets associated with a list of objects. They are known as video tags. They are stored for further reference in a CSV file.

Step 4

Here we determine a similarity matrix of dimension: Query Clusters x DB cluster numbers. Using Euclidean distance the cluster correspondence is obtained for each query cluster.

Based on a pre-determined threshold (Threshold > 15) the cluster is labeled to be a similar or dissimilar match.
